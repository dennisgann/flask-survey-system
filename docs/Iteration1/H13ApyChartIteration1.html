Object Oriented (OO) Design

SOLID Principles:

  • SRP - Single Responsibility Principle: low coupling between classes (if any), every class has unique responsibility (ie. only one reason to change)
  • OCP - Open Closed Principle: derived/sub. classes (abstraction and inheritance) e.g. Questione extends Object
  • DIP - Dependency Inversion Principle: dependencies are injected

MVC:

  • Model: models.py (Object, ObjectStore, etc.)
  • View: templates/ (*.html files for different pages)
  • Controller: routes.py & functions.py
